# 💰 Expense Tracker App

A Flutter-based expense tracking application with Firebase authentication, local SQLite storage, receipt photo capture, daily reminders, and cloud backup.

## 📱 Demo Video

[Demo Video](https://drive.google.com/drive/folders/1mfSx4y6yDaZRsxff_1EhluF3xDdOYfEh?usp=sharing)

## Overview

This project contains a simple expense tracker built with Flutter. The app lets a user:

- sign in with email and password, or register automatically if the account does not exist
- add, edit, and delete expense transactions
- create, edit, and delete categories
- take a photo of a receipt using the camera
- view expense summary charts
- back up expense data to Firebase Firestore
- upload receipt images to Firebase Storage when available
- receive a daily reminder notification

## Main Features

### Authentication
- Uses `firebase_auth`
- Login screen accepts email and password
- If sign-in fails, the app tries to create a new account with the same credentials
- After success, the app opens the home screen

### Expense Management
- Add new transactions from a bottom sheet form
- Edit existing transactions by tapping an item
- Delete transactions from the list
- Store transaction data locally in SQLite
- Show total spending at the top of the home screen
- Display a pie chart summary using `fl_chart`

### Category Management
- Add new categories
- Edit existing categories
- Delete categories
- Categories are stored locally in SQLite and joined with transactions for display

### Receipt Photo Capture
- The form supports receipt capture from the camera
- Selected image path is stored with the transaction
- If a receipt exists, it can be uploaded to Firebase Storage during backup

### Cloud Backup
- After a transaction is added or updated, the app runs a cloud backup
- Backup data is stored in Firestore under the `backups` collection with the user UID as the document ID
- The backup payload contains:
  - `transactions`
  - `updatedAt`
- Receipt images are uploaded to Firebase Storage and the download URL is stored in the backup payload when upload succeeds
- Deleting a transaction is only local in the current code path and is not followed by a cloud sync

### Notifications
- The app initializes local notifications on startup
- It requests notification permission on Android 13+
- A daily reminder is scheduled automatically
- The scheduled reminder time in the current code is **20:00**
- Timezone is detected with a fallback to `Asia/Jakarta`

## Data Flow

### Local Database
- `DatabaseService` opens a SQLite database named `expenses_<uid>.db`
- Transactions and categories are stored locally
- Transaction records include:
  - title
  - amount
  - image path
  - category ID

### Cloud Backup
- When the user saves or updates a transaction, the app builds a backup payload
- Each transaction can include a receipt image upload to Firebase Storage
- The backup is written to Firestore in the `backups/<uid>` document

## Configuration

### Firebase Setup
1. Create a Firebase project
2. Add an Android app to the Firebase project
3. Place `google-services.json` in `android/app/`
4. Make sure Authentication, Firestore, and Storage are enabled in the Firebase console

### Android Permissions Declared in the Manifest
The current Android manifest declares these permissions:

- `POST_NOTIFICATIONS`
- `RECEIVE_BOOT_COMPLETED`
- `SCHEDULE_EXACT_ALARM`
- `USE_EXACT_ALARM`
- `INTERNET`

## Current Limitations

These notes are based on the current code in this repository:

- The reminder time is fixed in code at **20:00**
- There is no UI to change the reminder time yet
- There is no test-notification button in the current implementation
- Backup is triggered automatically after add/update, not from a separate backup button
- Deleting a transaction does not currently trigger a cloud delete
- The app is effectively Android-focused in practice because the active implementation uses Android-specific notification and image-picker flows
- `firebase_options.dart` throws for iOS, so iOS is not configured in this project
